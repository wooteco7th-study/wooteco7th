- [ ]  단계 1 : 평일 비상 근무 순번
- [ ]  단계 2: 휴일 비상 근무 순번

    - 휴일 : 토요일, 일요일, 법정공휴일
- 입력/ 출력 형식

- 예외
    - [ ]  올바르지 않은 입력 형식의 경우
    - 순번
        - [ ]  순번 내에 중복 요소가 있는 경우
        - [ ]  **평일 순번, 휴일 순번에 각각 1회 편성** 되지 않은 경우

    - [ ]  닉네임

        - [ ]  최소 `5`명의 근무자
        - [ ]  최대 `35`명의 근무자

    - 순번 내에 중복요소가 있으면 안된다.

## 근무 순번 계산하기

- [ ]  단계 1 : 근무 순번 계산하기
- [ ]  단계 2: 근무 순번 해당 달의 1일부터 마지막일까지 출력하기
- 케이스 :
    - [ ]  연속 2일 근무할 경우 **같은 요일의 다음 근무자와 순서를 바꿔 편성**
    - [ ]  **앞의 날짜부터 순서를 변경**
- 입력/ 출력 형식
    - [ ]  기본 : `5월 1일 월 준팍`
    - [ ]  **평일**이면서 **법정공휴일**의 경우에만 요일 뒤에 **(휴일)** 표기
        - `5월 5일 금(휴일) 루루`

### 비상 근무일

- **순번**에 따라 비상 근무일을 배정
- 비상 근무자는 **연속 2일은 근무할 수 없다.**
    - 순번상 특정 근무자가 연속 2일 근무하게 되는 상황에는, **다음 근무자와 순서를 바꿔 편성**한다.
    - 다음 근무자의 경우 **같은 요일(휴일, 평일)의 바로 다음 근무자**이다.

```
- 평일 순번: 준팍,도밥,고니,수아,루루,글로,솔로스타,우코,슬링키,참새,도리
- 휴일 순번: 수아,루루,글로,솔로스타,우코,슬링키,참새,도리,준팍,도밥,고니
- 근무 예시: 준팍(월요일),도밥(화요일),고니(수요일),수아(목요일),루루(금요일/휴일),수아(토요일/휴일),...

```

- 비상 근무자 배정 시 다음 근무자와 순서를 바꿔야 하는 경우에는, **앞의 날짜부터 순서를 변경**해야 한다.

### WorkScheduler - 비상 근무표 생성기

- 필드
    - 평일 순번 - `Turn`
    - 휴일 순번 - `Turn`
- 기능
    - **순번**에 따라 비상 근무일을 배정
    - **연속 2일은 근무할 수 없도록 배정**
        - **다음 근무자와 순서를 바꿔 편성**

## 예외 메세지

- [ ]  `유효하지 않은 입력 값입니다. 다시 입력해 주세요.`

### 배경

- <우택호> 배달 주문 서비스
- 서버 개발
- **효율적으로 인력을 배치**하여 장애를 탐지하고 신속히 대응할 수 있을지
- 경기 일정마다 개발자를 배치
    - 당시에는 비상 대응 인력이 부족해서 경기 일정마다 같은 인원이 반복적으로 대기
    - 이제는 비상 대응 인력이 확보
- '월별 비상근무표'를 편성

### 입력 기능

- 월, 시작요일 입력 받기
- 평일 비상 근무 순서를 입력
- 휴일(토요일, 일요일, 법정공휴일) 비상 근무 순서를 입력
- [ ]  재입력
    - [ ]  사용자가 잘못된 값을 입력할 경우 IllegalArgumentException를 발생시키고, `"[ERROR]"`로 시작하는 에러 메시지를 출력 후 **해당 부분부터 다시 입력을 받는다.**
    - [ ]  재입력 위치
        - [ ]  월, 시작요일 : 다시 요청 메세지 출력 후 그 부분부터
        - [ ]  평일 순번, 휴일 순번 : **평일 순번부터 다시 입력**

### 출력 기능

- **평일**이면서 **법정공휴일**의 경우에만 요일 뒤에 **(휴일)** 표기
- 비상 근무표 출력을 완료하면 프로그램은 종료
- [ ]  재입력
    - [ ]  OO 부분부터 출력
- [ ]  프로그래밍 실행 결과 예시와 동일하게 입력과 출력이 이루어져야 한다.
